分享一下自己的工具库

在 js 中我们通常都会封装一些工具函数来放在项目的 utils 文件夹下，用来解决一些重复的工作。工具函数越写越多，每次新建一个项目时都会拷贝，之前编写的工具函数一直是自己使用，没有进行仔细的测试，所以在新项目中会不断完善，这样就有一个问题，一次修改别的地方也要找到并修改。

这样做效率低，且不断拷贝会占用一些不必要的空间。

为了偷懒，我将所用工具函数总结整理在一个文件夹内，并使用 Rollup 打包，然后发布到 npm 上。

这样使用就比较简单了，fxl_utils的github仓库。

如果你疑问为什么不使用大名鼎鼎的 lodash 时我只能说，目前我处于学习阶段工具函数出现的过程对于我来说也十分重要。

简介

这个工具库里面包含很多有一些常用的工具函数，例如获取数据类型的 getTypeOf ,实现了基础功能的防抖函数 debounce 和节流函数 throttle ,还有对数据进行判空的 isEmpty 。

当然更多的是基于我当时项目而编写的工具函数，有对数组和对象操作的，还有关于本地缓存的，也有实现基本功能的经典的发布订阅。

下面我从我开发中遇到的一些情况对几个工具函数做一个介绍。

场景一

在做后台管理时的菜单是有服务器传递过来的，我在编写服务器时将数据转成了树形数组方便前端使用，这是方便了，但是我同时要获取每个菜单下的权限来判断登录者的权限是否能进行相应的操作，为了方便多处使用，我编写了 findTreeArr 函数。

这个函数是递归的取属性，会过滤空值。

场景二

同样是管理系统中，我有一个导出员工的功能，能够导出全部也能根据部门筛选导出。这个导出分类是在前端实现的，因为这个项目主要在前端所以大部分功能都放在了前端。

我封装了一个工具函数 arrClassify 用于分类，还能传入工厂函数控制返回格式。

场景三

在给部门绑定菜单权限时，我的设计是将菜单编号序列化然后存在数据库中，这样每次修改时都是替换，所以要将两次的菜单编号的数组找到删除的，新添加的。

这个工具函数 arrContrast ,用来查找。

场景四

在这个项目中我没有使用 ORM框架 而是直接利用 mysql2 配合 SQL 语句进行的数据库操作。

这样的做法可以帮助我熟悉 SQL 语句和利用 node 操作数据库，但是如果全部使用手写 SQL 效率较低，所以我封装了关于增删改查语句的工具函数，在工具库中用了 sql 导出。

场景五

场景四对于 SQL 语句基础的封装，只能完成基础的单表查询，为了更灵活，我封装了 sql.execute 函数。

这个函数能够配合 handelFuzzyQuery 进行一些更简便的操作，但是该函数有一些使用限制，具体查看说明文档。

总结

以上是我挑了一些具有特色的工具函数，更多详情请查看说明文档。

另外场景中所提及的业务有些设计有缺陷，勿喷。
